Configure an AIO TDM module

Description

AIO-T configuration constants and parameters.

To configure an AIO-T module, call AIO_setModuleIntParameter to set one of the parameters listed below. All of the AIO-T module parameters are integers. You can also use AIO_getModuleIntParameter to read the current value of a parameter.

void configureAIOTModule()
{
//
// Read the AIO-T module firmware version
//
int firmwareVersion = 0;
int status = AIO_getModuleIntParameter(1, AIO_tdmParameterFirmwareVersion, &firmwareVersion);
if (status != AIO_OK)
{
return;
}
//
// Set 64 bits per TDM frame
//
status = AIO_setModuleIntParameter(1, AIO_tdmParameterBitsPerFrame, 64);
if (status != AIO_OK)
{
return;
}
std::cout << "AIO-T firmware version: " << firmwareVersion << std::endl;
}

Python example for AIO-TDM module configuration

Below is an example shows how to configure an AIO-TDM module using the EchoAIOInterface.dll.
API calls in this example make use of the Python API interface. The Python API interface is a wrapper around the C++ API interface. The Python API interface is provided as a convenience for Python programmers. It exposes the enums and defines from the C++ API interface

First, download the Python API interface files at https://github.com/mattgonzalez/EchoAIOExample
To use the inteface with the AIO Combo Module API, copy the file EchoAIOInterface_ComboModule_Python.py to the same directory as the python file, and import the file into your script.

#
# ctypes provides C compatible data types, and allows calling functions in DLLs or shared libraries.
#
import ctypes
import EchoAIOInterface_TDMModule_Python as tdm
#
# Change the following path to the location of EchoAIOInterface.dll.
# The default location is C:\Progam Files\Echo AIO\CLI + API\EchoAIOInterface.dll
#
path = 'C:\Program Files\Echo AIO\CLI + API\EchoAIOInterface.dll'
echo = ctypes.WinDLL(path)
#
# Define the ctypes function arguments and return types
#
echo.AIO_getModuleIntParameter.argtypes = [ctypes.c_int ,ctypes.c_int, ctypes.POINTER(ctypes.c_int)]
echo.AIO_getModuleIntParameter.restype = ctypes.c_int
echo.AIO_setModuleIntParameter.argtypes = [ctypes.c_int, ctypes.c_int, ctypes.c_int]
echo.AIO_getModuleDoubleParameter.argtypes = [ctypes.c_int, ctypes.c_int, ctypes.POINTER(ctypes.c_double)]
echo.AIO_getModuleDoubleParameter.restype = ctypes.c_int
#
# Initialize the EchoAIOInterface library
#
echo.AIO_initialize()
#
# Read the AIO-T module firmware version
#
firmwareVersion = ctypes.c_int()
echo.AIO_getModuleIntParameter(1, tdm.AIOTModuleParameters.AIO_tdmParameterFirmwareVersion, ctypes.byref(firmwareVersion))
print(hex(firmwareVersion.value))
#
# Set 64 bits per TDM frame
#
echo.AIO_setModuleIntParameter(1, tdm.AIOTModuleParameters.AIO_tdmParameterBitsPerFrame, 64)
#
# Shutdown the EchoAIOInterface library
#
echo.AIO_shutdown()

AIO_tdmParameterFirmwareVersion 

Read-only integer parameter for the AIO-T module firmware version; reads -1 on error.

AIO_tdmParameterBitsPerWord 

Sets the number of bits per TDM Word; 01 - 24 bits/word, 10 - 32 bits/word.

AIO_tdmParameterBitsPerFrame 

Sets the number of bits per TDM Frame; 01 - 64 bits/frame(not implemented), 10 - 128 bits/frame(not implemented), 11 - 256 bits/frame.

AIO_tdmParameterFSYNCPhaseDelay 

[Clock source mode] Integer parameter that determines whether FSYNC clocks out along with data or is delayed by 1/2 SCLK cycle; 0 - FSYNC clocks out along with data, 1 - FSYNC is delayed by 1/2 SCLK cycle

[Clock sink mode] Integer parameter that determines whether INPUT is delayed by 1 / 2 SLCK cycle; 0 - INPUT is sampled normally, 1 - INPUT sampling is delayed by 1 / 2 SCLK cycle

AIO_tdmParameterInvertSCLK 

Integer parameter to invert the SCLK signal; 0 - to Data and FSYNC clocks out on the falling edge of SCLK, 1 - Data and FSYNC clocks out on the rising edge of SCLK.

AIO_tdmParameterAudioDataShiftEnabled 

[Clock source mode] Integer parameter that delays the sampling of input SHIFT bits; 0 - TDM input sampling is aligned with TDM input, 1 - TDM output is advanced "SHIFT" bits adead of the TDM input.

[Clock sink mode] Integer parameter that enables SCLK output on BNC connector(Only valid for versions prior to 2.01. Must be set for versions 2.01 thru 2.0e, ignored for 2.0f and above)

AIO_tdmParameterClockMode 

Integer parameter that set the number of bits to delay INPUT (clock source mode) or advance OUTPUT (clock sink mode) when SHIFT_EN is set. (7 max, 0 is interpreted as 1 for backward compatibility)

AIO_tdmParameterAudioDataShiftBits 

Sets the AIO-T module to clock source or clock sink mode; 0 - clock source, 1 - clock sink.

AIO_tdmParameterLogicLevel 

Sets teh logic level for the TDM data; 0 - TDM data is active low, 1 - TDM data is active high.

AIO_tdmParameterFSYNCPosition 

8 Bit parameter that sets the position for start of positive portion of FSYNC. Valid positions are 0

AIO_tdmParameterFSYNCWidth 

8 bit integer parameter that sets the bit position for the start of positive portion of FSYNC. Valid positions are 0 (first data bit position) through frame length - 1

[clock source mode] bit 0 corresponds to the MSB of TDM output. [clock sink mode] bit 0 corresponds to the MSB of TDM input
